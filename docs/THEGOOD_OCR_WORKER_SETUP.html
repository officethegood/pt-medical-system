1สิ่งที่ต้องเตรียมก่อนเริ่ม

รายการคำอธิบาย
Google accountสำหรับสร้าง Gemini API key (ฟรี)
Cloudflare accountสมัครฟรีที่ cloudflare.com (ไม่ต้องใช้ credit card)
Node.jsติดตั้ง Node 18+ (ใช้รัน wrangler CLI)
TerminalPowerShell / cmd / bash

2ขอ Gemini API Key

Step 2.1 — เปิด Google AI Studio
  1. ไป aistudio.google.com/apikey
  2. Login ด้วย Google account
  3. กด Create API key
  4. เลือก / สร้าง Google Cloud project ใหม่ (เช่น thegood-ocr)
  5. Copy key ที่ได้ (ขึ้นต้น AIzaSy...) เก็บไว้ที่ปลอดภัย
⚠️ Security
  • API key นี้ ห้าม commit ลง Git
  • ห้าม paste ในแชต/อีเมล/Slack
  • จะอยู่ใน Cloudflare Worker เป็น encrypted secret เท่านั้น

3สร้าง Cloudflare Account

Step 3.1 — สมัคร Cloudflare
  1. ไป dash.cloudflare.com/sign-up/workers-and-pages
  2. กรอก email + password → verify email
  3. เลือก Free plan (ไม่ต้องใส่ credit card)

4ติดตั้ง Wrangler CLI

Step 4.1 — ติดตั้ง Node.js (ถ้ายังไม่มี)
  1. ไป nodejs.org → download LTS version
  2. ติดตั้ง (เลือก default options)
  3. เปิด PowerShell / Terminal ใหม่ → ทดสอบ: 
    node --version
npm --version
    ควรขึ้น v18.x ขึ้นไป
Step 4.2 — ติดตั้ง wrangler
npm install -g wrangler

(Windows: อาจต้อง run PowerShell as Administrator)

Step 4.3 — Login
wrangler login
  1. Browser จะเปิด Cloudflare login page
  2. Login → Authorize Wrangler
  3. กลับมาที่ terminal → ควรเห็น "Successfully logged in"

5Deploy Worker

Step 5.1 — Clone หรือ Download repo

ถ้ามี Git installed:

git clone https://github.com/officethegood/pt-medical-system.git
cd pt-medical-system/cloudflare

หรือ download ZIP จาก GitHub แล้ว extract → เข้าโฟลเดอร์ cloudflare/

Step 5.2 — ตั้ง Secret (Gemini API Key)
wrangler secret put GEMINI_API_KEY

เมื่อถาม → paste Gemini API key (AIzaSy...) → กด Enter

ผล: ✨ Success! Uploaded secret GEMINI_API_KEY

Step 5.3 — Deploy
wrangler deploy

จะได้ผลลัพธ์ประมาณ:

Total Upload: 2.34 KiB / gzip: 0.98 KiB
Uploaded thegood-ocr-proxy (2.15 sec)
Deployed thegood-ocr-proxy triggers (0.37 sec)
  https://thegood-ocr-proxy.<your-subdomain>.workers.dev

Copy URL ที่ได้ — จะเอาไปใส่ใน webapp ต่อ

6ทดสอบ Worker

Step 6.1 — Test ง่ายๆ จาก terminal
curl -X POST https://thegood-ocr-proxy.<your-subdomain>.workers.dev ^
  -H "Content-Type: application/json" ^
  -d "{\"contents\":[{\"parts\":[{\"text\":\"Hello\"}]}]}"

ควรได้ response JSON จาก Gemini พร้อม generated text

ถ้าได้ error:

Errorสาเหตุ + แก้
500: missing GEMINI_API_KEY secretยังไม่ได้ตั้ง secret → ทำ Step 5.2 อีกครั้ง
403: Origin not allowedถ้าตั้ง ALLOWED_ORIGINS → ลบออกตอน dev (ดู Step 8)
400: Invalid request bodyJSON format ผิด

7เปลี่ยน URL ใน PT Medical System

Step 7.1 — แก้ config.js

เปิดไฟล์ shared/config.js ใน TheGood repo:

OCR_PROXY_URL: 'https://thegood-ocr-proxy.<your-subdomain>.workers.dev'

Commit + push:

git add shared/config.js
git commit -m "feat(thegood): switch OCR to own Cloudflare Worker"
git push
Step 7.2 — รอ GitHub Pages deploy 1-2 นาที

ทดสอบ: เปิด TheGood webapp → Transport → สร้างเคสใหม่ → ถ่าย/อัพโหลดภาพบัตร ปชช. → กด OCR

✅ ถ้าได้ข้อมูลบัตรออกมา = Worker ของ TheGood ทำงานแล้ว

8Production Hardening (แนะนำ)

Step 8.1 — จำกัด Origin

แก้ cloudflare/wrangler.toml:

[vars]
ALLOWED_ORIGINS = "https://officethegood.github.io"

Deploy ใหม่:

wrangler deploy

หลังจากนี้ มีเฉพาะ webapp TheGood ที่เรียก worker ได้

Step 8.2 — ตั้ง Gemini quota/budget
  1. ไป console.cloud.google.com/billing/budgets
  2. สร้าง budget alert เช่น $5/เดือน → email alert
  3. ป้องกัน bill แพงถ้าโดน abuse

10Troubleshooting

อาการแก้
wrangler: command not found ยังไม่ได้ติดตั้ง wrangler → ทำ Step 4.2 / เปิด terminal ใหม่
Deploy ช้า/timeout ลอง wrangler deploy --verbose ดู error — บางครั้ง network issue แค่รันใหม่
OCR ยังใช้ Supwilai's worker ยังไม่ได้แก้ config.js หรือยังไม่ refresh → F5 webapp
Gemini ตอบ 429 Too Many Requests Free Gemini = 15 req/min — ใช้ถี่เกินต้อง upgrade หรือรอ
CORS error ALLOWED_ORIGINS ไม่ตรง — ตรวจว่าโดเมน webapp ตรงกับที่ตั้งไว้
